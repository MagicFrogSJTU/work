## 第9页 心得2-1：数据定义，是算法问题

页内文字：

- 输入输出，定义了"你要达到什么目的"，即任务本身。
- 评测指标，定义了”你认为什么是好的”，即任务的优化目标。
- 抛开这三个谈算法，是无源之水，镜中之月。

图表：

```mermaid
flowchart LR
    A[输入] --> D[任务定义]
    B[输出] --> D
    C[评测指标] --> D
    D --> E[好的设计是成功的一半]
```

## 第10页 心得2-2：数据闭环，是工程问题

页内文字：

- 数据理解：标注、统计、可视化、结构化
- 数据策划：挖掘、采集、合成、平衡
- 人工标注：
  - 质量重于数量. scaleAI/surgeAI标注员时薪可至200美元.
  - 成本高昂
    - 基于不信任的团队管理：规格/产线管理/工艺/质量
    - 基础设施：标注平台，人机共标算法，统计/运维后台

图表：

```mermaid
flowchart LR
    subgraph H[数据策划]
        C(存量-原始数据池) -- 挖掘 --> A[数据理解]
        G(流式数据管道) -- 采集 --> A[数据理解]
        E[数据合成] --> F
        F(结构化数据池) --> B[数据平衡]
        A[数据理解] --> F
    end
    B --> D[模型训练]
    D --> H
```

## 第11页 心得2-3：训练数据没有量，就没有真正的价值

页内文字：

- 支撑一个商用SOTA端到端智驾模型，需要1000万以上个数据片段，数据存储规模超过100PB。
- 排除数据需求量大的预训练阶段，后训练阶段的训练数据需求量也在十万级以上。
- 支撑这一切的，需要有强大的大数据基础设施。
