## 大纲

建议不要只按年份平铺，而是围绕一条主线展开：自动驾驶算法从模块化感知，走向端到端，再走向世界模型和智驾大模型；而我个人的角色，也从核心算法研发，逐步升级为系统建设者和团队负责人。

每一章建议都按同一结构来讲，这样内容会更稳，也更容易做PPT：

1. 这一阶段行业和业务要解决什么问题
2. 当时主流技术架构是什么，为什么会演进
3. 华为/团队选择了什么路线
4. 我具体负责了什么，解决了什么关键问题
5. 这一阶段沉淀了什么能力，并如何导向下一阶段

按这个思路，汇报大纲可以改成：

1. 个人简介：我的技术主线与能力画像
   - 5年自动驾驶研发经历，主线始终围绕“算法 + 数据闭环 + 工程系统”
   - 角色变化：核心算法研发 -> 项目负责人 -> 大模型数据组负责人
   - 一句话总结我的核心优势：既能做前沿算法，也能把算法真正做成可量产、可闭环、可持续迭代的系统

2. 自动驾驶算法的第一阶段：从传统感知到 3D 感知重构（2021-2024）
   - 业务背景：智能驾驶从高速场景走向更复杂城区场景，传统2D感知 + 规则后处理逐渐遇到瓶颈
   - 旧架构：2D image-based + 3D point cloud-based + rule-based post-processing
   - 新架构：以3D BEV为核心的统一空间表征，叠加 occupancy 通用障碍物感知和轻地图/众包地图能力
   - 关键技术问题：如何从“检测结果拼接”走向“统一空间建模”，如何兼顾白名单目标和开放世界障碍物
   - 我主要做的工作：
     - 参与ADS1.0、ADS2.0两代静态环境感知系统研发，覆盖离线重建、在线BEV/MapTR算法、车端C++节点对齐
     - 负责轻地图前融合方案预研，推动去高精地图路线落地
     - 进行SpatialRNN等隐式地图表达预研，探索在线地图建模的新范式
     - 设计新一代环境感知中的结构化建模方案，支撑能力从高速扩展到城区和园区
   - 这一阶段的核心积累：
     - 对自动驾驶模块化系统和感知架构演进有系统理解
     - 能把单点算法问题，抽象成整条链路上的系统问题
     - 开始形成“算法效果依赖数据闭环和工程基础设施”的认识

3. 自动驾驶算法的第二阶段：从单点模型到端到端统一建模（2024-2026）
   - 业务背景：模块化方案虽然可解释、可分工，但链路长、误差传递重，越来越难继续逼近人类驾驶决策
   - 架构变化：从感知-预测-规划串联，走向感知-预测-规划一体化；从多模块接口设计，走向基于Transformer的统一表示学习
   - 关键范式变化：
     - 从“识别世界”转向“直接为驾驶决策建模”
     - 从监督式模块优化，转向大规模数据驱动的统一建模
     - diffusion-based action policy generation 与传统 next-token 生成有本质不同，更适合连续控制和多峰动作分布
   - 我主要做的工作：
     - 带领团队建设智驾大模型数据系统，为下一代自动驾驶预研项目提供数据能力底座
     - 建设多模态自动化打标算法库，利用LoRA、多模态RAG、Prompt优化等方法提升智驾垂域数据生产能力
     - 建设图片/视频结构化和非结构化检索能力，支撑千万级CLIP数据召回和精排探索
     - 建设自动化数据闭环产线，形成VQA/Caption预训练数据、CoT长文本SFT数据等多类数据资产
   - 这一阶段的核心积累：
     - 我从算法研发进一步走向“模型能力 x 数据能力 x 系统能力”的统一建设
     - 对多模态数据、训练数据组织、模型迭代机制有了更完整的理解
     - 开始从“做模型”转向“做支撑模型持续进化的平台”

4. 自动驾驶算法的第三阶段：世界模型、强化学习与智驾大模型（2026-）
   - 为什么端到端之后还不够：真实驾驶不仅要做短时动作生成，还要具备长期推演、反事实分析、自我改进和复杂交互理解能力
   - 世界模型：学习环境演化规律，构建可预测未来的内部模拟器
   - 强化学习：让系统通过反馈不断改进策略，实现更强的自我学习能力
   - 智驾大模型：引入慢思考、链式推理、辅助任务分解、多模态理解与生成能力
   - 一个更大的判断：LLM、生成模型、世界模型、强化学习的方法论正在逐渐统一，自动驾驶会成为最重要的具身智能落地方向之一
   - 我主要关注和要继续投入的方向：
     - 如何构建高质量的世界模型训练数据和评估体系
     - 如何把生成式方法真正用于智驾决策，而不只是辅助感知或数据生产
     - 如何把数据闭环升级为“模型自我进化闭环”
   - 这一阶段想传达的观点：未来的自动驾驶竞争，核心不只是单个模型精度，而是世界建模能力、学习效率和系统闭环效率的竞争

5. 贯穿5年的核心方法论：我真正积累了什么
   - 第一，不把算法看成孤立模型，而是看成“数据-模型-系统-业务闭环”的整体
   - 第二，不只追求论文式指标，而是关注可量产、可迭代、可工程化落地
   - 第三，面对范式迁移时，持续从旧问题里抽象新方法，而不是只追热点
   - 第四，从个人贡献走向组织贡献，最终沉淀成团队能力和基础设施能力

6. 结束页：为什么我适合下一阶段的AI/自动驾驶工作
   - 我经历过自动驾驶从模块化感知到端到端、再到大模型化的关键演进阶段
   - 我既做过核心算法，也做过数据闭环、基础设施和团队建设
   - 我能够把前沿AI方法和复杂业务场景结合起来，推动真正可落地的系统演进
